# Выводы по сравнению методов логического кодирования

## Формулы для расчёта частот

Исходные данные: сообщение 24 бита (C3 C4 C2), скорость передачи C = 100 Мбит/с

### Основные формулы

- **Максимальная частота**: f_в = C / 2 (для двухуровневых сигналов NRZ, 4B/5B, скремблирование)
- **Максимальная частота (Manch)**: f_в = C (для манчестерского кодирования)
- **Минимальная частота**: f_н = C / (2 × N_max), где N_max — максимальная серия нулей
- **Требуемая полоса**: Δf = f_в - f_н

### Расчёт для методов

| Метод | f_в (МГц) | f_н (МГц) | Δf (МГц) | Примечание |
|-------|-----------|-----------|----------|------------|
| NRZ | 100/2 = **50** | 100/(2×4) = **12.5** | 50 | N_max = 4 |
| Manchester | 100 = **100** | 100/2 = **50** | 100 | Переход каждый бит |
| RZ | 100 = **100** | 100/(2×2) = **25** | 100 | N_max = 2 |
| AMI | 100/2 = **50** | 100/(2×5) = **10** | 50 | N_max = 5 |
| 4B/5B+NRZ | 125/2 = **62.5** | 125/(2×4) = **15.6** | 62.5 | 125 Мбит/с, N_max = 3 |
| Scrambled NRZ | 100/2 = **50** | 100/(2×3) = **16.7** | 50 | N_max = 3 |

## Сравнение 4B/5B и скремблирования

| Параметр | 4B/5B + NRZ | Scrambled NRZ |
|----------|-------------|---------------|
| Полоса пропускания | 62.5 МГц | 50.0 МГц |
| Избыточность | 20% | 0% |
| Макс. серия нулей | 3 | 3 |
| Постоянная составляющая | Нет | Нет |

## Лучший метод: скремблирование

**Обоснование:**

1. **Полоса пропускания**: Scrambled NRZ требует на 20% меньшую полосу (50 МГц vs 62.5 МГц)

2. **Избыточность**: Скремблирование не добавляет битов (0% избыточность), в то время как 4B/5B увеличивает скорость на 20% (с 100 до 125 Мбит/с)

3. **Эффективность устранения длинных последовательностей**: Оба метода сокращают макс. серию нулей до 3

4. **Отсутствие DC**: Оба метода устраняют постоянную составляющую

## Итоговая рекомендация

Для данной задачи **скремблирование** является оптимальным выбором, так как:
- Не требует дополнительной полосы пропускания
- Не увеличивает скорость передачи
- Эффективно устраняет длинные последовательности нулей

4B/5B целесообразно применять когда требуется гарантированная синхронизация и допустимо увеличение скорости канала.
