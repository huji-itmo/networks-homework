# SUMMARY.md — Техническая документация

## Обзор проекта

Это библиотека для работы с сигнальными кодировками в компьютерных сетях. Используется в курсе "Компьютерные сети" для:
- Физического кодирования сигналов (NRZ, Manchester, RZ, AMI)
- Блочного кодирования (4B/5B)
- Скремблирования
- Расчёта метрик и генерации отчётов

## Алгоритмы кодирования

### NRZ (Non-Return-to-Zero)

**Принцип работы:** Уровень сигнала остаётся постоянным в течение всего бита. Бит 1 кодируется высоким уровнем, бит 0 — низким.

**Реализация** (`encodings.py:19-37`):
- Каждый бит преобразуется в соответствующий уровень
- По умолчанию: 1 → +1, 0 → -1
- Поддерживаются произвольные уровни через параметр `levels`

**Характеристики:**
- Полоса: от C/(2×nmax) до C/2 МГц (C — скорость в Мбит/с)
- Уровней сигнала: 2
- Имеет постоянную составляющую (DC)
- Проблема: длинные последовательности нулей/единиц приводят к потере синхронизации

---

### Manchester

**Принцип работы:** Обязательный переход в середине каждого бита. Направление перехода определяет значение бита.

**Варианты** (`encodings.py:40-68`):
- **IEEE 802.3**: 0 = LOW→HIGH, 1 = HIGH→LOW
- **Thomas**: инвертированная кодировка

**Реализация:**
- Каждый бит генерирует 2 полубита (2 выборки на бит)
- 1 → [1, -1], 0 → [-1, 1] (для IEEE)

**Характеристики:**
- Полоса: от C/2 до C МГц (в 2 раза шире чем NRZ)
- Уровней сигнала: 2
- Нет постоянной составляющей
- Гарантированная синхронизация благодаря переходу в каждом бите

---

### RZ (Return-to-Zero)

**Принцип работы:** Сигнал возвращается к нулю во второй половине битового интервала.

**Реализация** (`encodings.py:71-89`):
- 1 → [level, zero]
- 0 → [zero, level]

**Характеристики:**
- Полоса: от C/4 до C МГц
- Уровней сигнала: 3 (низкий, ноль, высокий)
- Нет постоянной составляющей
- Хорошая синхронизация, но неэффективное использование полосы

---

### AMI (Alternate Mark Inversion)

**Принцип работы:** Биполярное кодирование с чередованием полярности единиц.

**Реализация** (`encodings.py:92-116`):
- 0 → нулевой уровень
- 1 → чередующиеся положительный и отрицательный уровни

**Характеристики:**
- Полоса: зависит от максимальной серии нулей
- Уровней сигнала: 3
- Нет постоянной составляющей (при сбалансированных данных)
- Проблема: длинные последовательности нулей

---

### 4B/5B

**Принцип работы:** Блочное кодирование — каждые 4 бита заменяются на 5-битный символ.

**Таблица преобразования** (`encodings.py:119-136`):
```
0000 → 11110
0001 → 01001
...
1111 → 11101
```

**Особенности:**
- Гарантирует не более 3 нулей подряд
- Избыточность: 20% (4 бита → 5 бит)
- Используется перед NRZ для улучшения синхронизации
- Не увеличивает полосу, но увеличивает скорость в канале

**Декодирование** (`encodings.py:168-188`):
- Обратная операция с проверкой валидности символов
- Неизвестные символы вызывают ValueError

---

### Скремблирование

**Что это и зачем нужно:**

Скремблирование (от англ. scramble — перемешивать) — это метод преобразования битовой последовательности，目的是 устранить длинные серии одинаковых битов (нулей или единиц). В отличие от блочного кодирования (4B/5B), скремблирование не добавляет избыточности — длина выходной последовательности равна длине входной.

**Проблемы, которые решает скремблирование:**

1. **Длинные последовательности нулей** — при NRZ кодировании длинные серии нулей приводят к отсутствию переходов сигнала, что затрудняет синхронизацию приёмника
2. **Постоянная составляющая (DC)** — длинные серии единиц создают постоянный уровень сигнала, что проблематично для систем с трансформаторной развязкой
3. **Эффект электромагнитных помех** — однообразный сигнал создаёт сильные электромагнитные излучения определённой частоты

**Линейная обратная связь (LFSR):**

Скремблирование основано на регистре сдвига с линейной обратной связью (Linear Feedback Shift Register). Это аппаратный примитив, широко используемый в телекоммуникациях.

В нашей реализации используется полином 7-го порядка:

```
B_i = A_i ⊕ B_{i-5} ⊕ B_{i-7}
```

Где:
- `A_i` — i-й бит исходного (нескремблированного) сообщения
- `B_i` — i-й бит скремблированного сообщения
- `⊕` — операция XOR (исключающее ИЛИ)
- `B_{i-5}`, `B_{i-7}` — биты на 5 и 7 позиций назад в выходной последовательности

**Как это работает на практике:**

Рассмотрим пример для последовательности `[1, 1, 0, 0, 0, 0, 1, 0]`:

```
i=0: B_0 = A_0 ⊕ 0 ⊕ 0 = 1 ⊕ 0 ⊕ 0 = 1
i=1: B_1 = A_1 ⊕ 0 ⊕ 0 = 1 ⊕ 0 ⊕ 0 = 1
i=2: B_2 = A_2 ⊕ 0 ⊕ 0 = 0 ⊕ 0 ⊕ 0 = 0
i=3: B_3 = A_3 ⊕ 0 ⊕ 0 = 0 ⊕ 0 ⊕ 0 = 0
i=4: B_4 = A_4 ⊕ 0 ⊕ 0 = 0 ⊕ 0 ⊕ 0 = 0
i=5: B_5 = A_5 ⊕ B_0 ⊕ 0 = 0 ⊕ 1 ⊕ 0 = 1
i=6: B_6 = A_6 ⊕ B_1 ⊕ B_0 = 1 ⊕ 1 ⊕ 1 = 1
i=7: B_7 = A_7 ⊕ B_2 ⊕ B_1 = 0 ⊕ 0 ⊕ 1 = 1
```

Результат: `[1, 1, 0, 0, 0, 1, 1, 1]`

**Дескремблирование:**

Операция обратима! Для восстановления исходных данных используется та же формула, но applied к скремблированным битам:

```
A_i = B_i ⊕ B_{i-5} ⊕ B_{i-7}
```

Это работает благодаря свойству XOR: если `B_i = A_i ⊕ X ⊕ Y`, то `A_i = B_i ⊕ X ⊕ Y`.

**Полиномы скремблирования:**

Разные стандарты используют разные полиномы:

| Стандарт | Полином | Применение |
|----------|---------|------------|
| x⁷ + x⁶ + 1 | B_i = A_i ⊕ B_{i-6} ⊕ B_{i-7} | Устаревшие |
| x⁷ + x⁵ + 1 | B_i = A_i ⊕ B_{i-5} ⊕ B_{i-7} | Наша реализация (широко используется) |
| x⁹ + x⁴ + 1 | B_i = A_i ⊕ B_{i-4} ⊕ B_{i-9} | ISDN |
| x¹⁵ + x¹⁴ + 1 | B_i = A_i ⊕ B_{i-14} ⊕ B_{i-15} | Стандарты ATM |

**Реализация** (`encodings.py:191-241`):

```python
def scramble(bits: list[int], poly_order: int = 7) -> list[int]:
    output = []
    for i, a in enumerate(bits):
        b_minus_5 = output[i - 5] if i >= 5 else 0
        b_minus_7 = output[i - 7] if i >= 7 else 0
        b = a ^ b_minus_5 ^ b_minus_7
        output.append(b)
    return output
```

Ключевые моменты реализации:
- Для первых 7 бит используются нули в качестве "исторических" значений
- Это создаёт переходный период, который необходимо учитывать при декодировании

**Характеристики:**

| Свойство | Значение |
|----------|----------|
| Избыточность | 0% (длина не меняется) |
| Обратимость | Да (зеркальная операция) |
| Синхронизация | Требуется (нуженknown начальный state) |
| Макс. серия нулей | Обычно ≤ 7 (зависит от полинома) |
| DC компонент | Устраняется при сбалансированных данных |

**Преимущества:**

1. **Нет избыточности** — скорость не меняется, полоса пропускания остаётся как у исходного NRZ
2. **Обратимость** — дескремблирование выполняется той же операцией
3. **Аппаратная реализуемость** — LFSR легко реализуется на сдвиговых регистрах
4. **Эффективность** — простая побитовая операция XOR

**Недостатки:**

1. **Требуется синхронизация** — приёмник должен знать начало последовательности
2. **Переходный период** — первые биты могут быть нестабильными
3. **Не является шифрованием** — тривиально обращается без ключа
4. **Зависимость от данных** — некоторые паттерны могут создавать длинные серии

**Сравнение с 4B/5B:**

| Характеристика | 4B/5B | Скремблирование |
|---------------|-------|-----------------|
| Избыточность | 20% | 0% |
| Макс. нулей | 3 | ≤7 (обычно) |
| Сложность реализации | Простая (таблица) | Простая (XOR) |
| Синхронизация | Не требуется | Требуется |
| Аппаратная реализация | Требует память | Требует регистры |

**Практическое применение:**

- **Fast Ethernet (100BASE-TX)** — использует MLT-3 + 4B/5B
- **Gigabit Ethernet (1000BASE-T)** — комбинирует PAM-5 + скремблирование
- **DSL технологии** — скремблирование для балансировки спектра
- **Цифровое телевидение** — устранение длинных серий нулей в потоке

## Метрики и анализ

### Расчёт метрик (`metrics.py`)

Модуль `metrics.py` вычисляет характеристики кодирования:

| Метрика | Описание |
|---------|----------|
| `bit_rate` | Скорость передачи (Мбит/с) |
| `channel_rate` | Скорость в канале (с учётом избыточности) |
| `f_high` | Верхняя граница частоты |
| `f_low` | Нижняя граница частоты |
| `bandwidth` | Требуемая полоса пропускания |
| `has_dc_component` | Наличие постоянной составляющей |
| `signal_levels` | Количество уровней сигнала |
| `max_consecutive_zeros` | Максимальная серия нулей |

### Формулы расчёта

**NRZ:**
- f_high = C/2
- f_low = C/(2×nmax), где nmax — макс. серия одинаковых битов

**Manchester:**
- f_high = C
- f_low = C/2 (гарантированный переход)

**AMI:**
- f_high = C/2
- f_low = C/(2×(nmax+1))

## Характеристики сигналов

При выборе метода кодирования необходимо учитывать ряд характеристик сигнала, которые влияют на качество передачи, требования к оборудованию и помехоустойчивость.

### Полоса пропускания (Bandwidth)

**Что это:** Полоса пропускания — это диапазон частот, который может пропустить канал связи без существенного затухания. Измеряется в герцах (Гц) или мегагерцах (МГц).

**Почему важно:** Чем шире полоса пропускания, тем выше потенциальная скорость передачи данных. Однако более широкая полоса означает:
- Большую стоимость оборудования
- Большие потери при передаче на большие расстояния
- Большую восприимчивость к помехам

**Связь скорости и полосы:** Для цифровых сигналов справедлива формула Найквиста:
```
C = 2B × log₂(M)
```

Где:
- C — максимальная скорость (бит/с)
- B — полоса пропускания (Гц)
- M — количество уровней сигнала

Для бинарного сигнала (M=2): C = 2B, то есть для передачи 100 Мбит/с требуется минимум 50 МГц полосы.

**Верхняя граница частоты (f_high):** Определяется максимальной скоростью изменений сигнала. Для NRZ это C/2, для Manchester — C (из-за переходов в середине каждого бита).

**Нижняя граница частоты (f_low):** Определяется минимальной частотой изменений, которая зависит от максимальной длительности непрерывного состояния. Чем длиннее серия одинаковых битов, тем ниже f_low.

```
f_low = C / (2 × nmax)
```

Где nmax — максимальное количество одинаковых битов подряд.

### Уровни сигнала (Signal Levels)

**Что это:** Количество различных уровней напряжения или тока, используемых для представления данных.

| Кодировка | Уровней | Значения |
|-----------|---------|----------|
| NRZ | 2 | -1, +1 (или 0, 1) |
| Manchester | 2 | -1, +1 |
| RZ | 3 | -1, 0, +1 |
| AMI | 3 | -1, 0, +1 |

**Влияние на помехоустойчивость:** Большее количество уровней позволяет передавать данные быстрее при той же полосе, но требует более точного оборудования и более чувствительно к шуму.

**Связь с пропускной способностью:** При фиксированной полосе пропускания увеличение уровней увеличивает скорость, но уменьшает расстояние устойчивой передачи.

### Постоянная составляющая (DC Component)

**Что это:** Постоянная составляющая — это среднее значение сигнала за длительный период. Если сигнал имеет DC компонент, его среднее значение не равно нулю.

**Проблемы с DC:**
1. **Трансформаторная развязка** — в большинстве сетей используется трансформаторная развязка для защиты от перенапряжений. Трансформаторы не пропускают постоянный ток, поэтому сигналы с DC компонентом искажаются.

2. **Электролиз** — в телефонных линиях DC компонент может вызывать электролиз и коррозию проводов.

3. **Смещение приёмника** — DC компонент может вызвать смещение нуля в приёмнике.

**Какие кодировки имеют DC:**
- NRZ — имеет DC компонент при несбалансированных данных
- Manchester — не имеет DC (гарантированные переходы)
- RZ — не имеет DC (возврат к нулю)
- AMI — не имеет DC при сбалансированных единицах
- Скремблированный NRZ — устраняет DC при сбалансированных данных

### Спектральная плотность мощности

**Что это:** Распределение мощности сигнала по частотам. Показывает, какую мощность сигнал имеет на каждой частоте.

**Почему важно:**
- Помогает предсказать помехи между каналами
- Определяет требования к фильтрам
- Влияет на электромагнитную совместимость (EMC)

**Характерные спектры:**

```
NRZ:        ████████████░░░░░░░░░
            (основная энергия на низких частотах)

Manchester: ██████████░░░░░░░░░░░
            (шире, но без DC)

RZ:         ██████████████████░░
            (основная энергия на высоких частотах)

AMI:        █████████░░░░░░░░░░░░
            (зависит от паттерна данных)
```

### Плотность переходов (Transition Density)

**Что это:** Процент битовых позиций, на которых происходит изменение уровня сигнала.

```
Transition Density = (количество переходов) / (всего битов - 1)
```

**Пример:**
- Последовательность `10101010` — плотность 100% (каждый бит меняется)
- Последовательность `11110000` — плотность 25% (только 3 перехода на 8 битов)

**Почему важно:**
- Высокая плотность переходов обеспечивает синхронизацию приёмника
- При низкой плотности приёмник может потерять битовую синхронизацию
- NRZ с длинными сериями нулей имеет плотность 0%

**Минимальная плотность:** Для надёжной синхронизации рекомендуется не менее 10-15% переходов.

### Максимальная серия нулей (Max Consecutive Zeros)

**Что это:** Наибольшее количество нулевых битов подряд в закодированном сигнале.

**Проблемы:**
1. **Синхронизация** — приёмник использует переходы для подстройки时钟 (clock recovery). Длинные серии нулей означают отсутствие переходов.

2. **AC coupling** — конденсаторы в цепях связи блокируют DC, и при длинных сериях нулей сигнал может "уплыть".

**Типичные ограничения:**
- Fast Ethernet (100BASE-TX): макс. 4 нуля (после 4B/5B)
- Gigabit Ethernet: макс. 24 нуля (после скремблирования)
- RS-232: ограничений нет, но рекомендуется не более 10

### Скорость в канале (Channel Rate)

**Что это:** Фактическая скорость передачи в физическом канале с учётом избыточности.

**Отличия от битовой скорости:**
- Битовая скорость (bit rate) — скорость исходных данных
- Канальная скорость (channel rate) — скорость в канале после кодирования

**Пример:**
- Исходные данные: 100 Мбит/с
- После 4B/5B: 125 Мбит/с (избыточность 20%)
- Manchester: 100 Мбит/с (скорость не меняется, но полоса удваивается)

### Синхронизация (Clock Recovery)

**Проблема:** Приёмник должен знать, когда начинается и заканчивается каждый бит. Для этого используется тактовый сигнал (clock).

**Решения в разных кодировках:**

| Метод | Способ синхронизации |
|-------|---------------------|
| NRZ | Внешняя синхронизация, проблема при длинных сериях |
| Manchester | Гарантированный переход в каждом бите |
| RZ | Переход в начале каждого бита |
| AMI | Требуется баланс нулей/единиц |
| 4B/5B | Ограничение нулей — улучшает синхронизацию |
| Скремблирование | Устраняет длинные серии |

### Сравнительная таблица характеристик

| Характеристика | NRZ | Manchester | RZ | AMI | 4B/5B | Scrambled |
|---------------|-----|------------|----|----|-------|-----------|
| Полоса (B) | C/2 | C | C | C/2 | C/2 | C/2 |
| Уровней | 2 | 2 | 3 | 3 | 2 | 2 |
| DC компонент | Да | Нет | Нет | Нет* | Да | Нет* |
| Мин. переходов | 0 | C | C | 0 | 3/5 | >0 |
| Избыточность | 0% | 0% | 0% | 0% | 20% | 0% |
| Сложность | Простая | Простая | Средняя | Средняя | Средняя | Простая |

*При сбалансированных данных

### Практические рекомендации по выбору кодировки

**Выберите NRZ если:**
- Ограничена полоса пропускания
- Есть внешняя синхронизация
- Простота реализации важна

**Выберите Manchester если:**
- Критична надёжная синхронизация
- Средства передачи не имеют DC (Ethernet 10BASE-T)
- Полоса пропускания не ограничена

**Выберите AMI если:**
- Нужен биполярный сигнал ( телефонные станции)
- Важна экономия энергии
- Есть трансформаторная развязка

**Добавьте 4B/5B если:**
- Нужно гарантировать синхронизацию (Fast Ethernet)
- Допустима избыточность 20%

**Добавьте скремблирование если:**
- Нужно устранить DC без избыточности
- Данные могут иметь длинные серии нулей
- Есть синхронизация начала последовательности

### Запуск тестов

```bash
pytest tests/ -v
```

### Структура тестов (`tests/`)

- `test_encodings.py`: Основные тесты кодирования
- `test_metrics.py`: Тесты расчёта метрик

### Принципы тестирования

1. **Тестирование граничных условий:**
   - Все нули, все единицы
   - Короткие последовательности (1 бит)
   - Смешанные паттерны

2. **Тестирование round-trip:**
   - encode → decode должен давать исходные данные

3. **Тестирование свойств:**
   - 4B/5B: не более 3 нулей подряд
   - Скремблирование: обратимость

## Использование API

### Кодирование данных

```python
from signal_plotter.encodings import (
    bytes_to_bits, nrz_encode, manchester_encode, 
    ami_encode, encode_4b5b, scramble
)

# Данные
data = b"\xB2\x69\x9C"
bits = bytes_to_bits(data)

# Физическое кодирование
signal_nrz = nrz_encode(bits)
signal_manchester = manchester_encode(bits, variant="ieee")

# Блочное кодирование
encoded_4b5b = encode_4b5b(bits)

# Скремблирование
scrambled = scramble(bits)
```

### Расчёт метрик

```python
from signal_plotter.metrics import (
    calculate_nrz_metrics, calculate_ami_metrics
)

metrics = calculate_nrz_metrics(bits, bit_rate=100.0)
print(f"Полоса: {metrics.bandwidth} МГц")
print(f"DC компонент: {metrics.has_dc_component}")
```

## Поддержка и развитие

### Добавление нового кодирования

1. Добавить функцию кодирования в `encodings.py`
2. Добавить функцию расчёта метрик в `metrics.py`
3. Добавить тесты в `tests/test_encodings.py`
4. Обновить `report_generator.py` если нужно

### Типичные проблемы

1. **Потеря синхронизации**: Использовать Manchester или 4B/5B
2. **Постоянная составляющая**: Использовать AMI, Manchester или скремблирование
3. **Длинные серии нулей**: Использовать 4B/5B или скремблирование

### Безопасность

Код не выполняет валидацию входных данных в некоторых местах:
- Функции `encode_*` принимают произвольные биты без проверки
- 4B/5B и дескремблирование проверяют валидность символов

**Рекомендации:**
- Не использовать для криптографии (скремблирование — не шифрование)
- Валидировать входные данные перед обработкой
- Обрабатывать исключения при декодировании
